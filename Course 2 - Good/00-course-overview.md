# TaskFlow — Learn React By Building a Real Project

## The Method

This is NOT a theory course. You will build ONE project — **TaskFlow** (a project management app) — that starts as a single-file todo list and grows into a full-featured app with auth, drag-and-drop, real-time updates, and a backend.

## How Each Lesson Works

```
┌────────────────────────────────────────────┐
│  LESSON STRUCTURE                           │
├────────────────────────────────────────────┤
│                                             │
│  1. CONCEPT (2 min read)                    │
│     One clear idea. No fluff.               │
│                                             │
│  2. CODE ALONG (10-15 min)                  │
│     Commands you run. Code you write.        │
│     Bold text = what you type                │
│                                             │
│  3. WHY THIS MATTERS (1 min)                │
│     Real-world connection.                  │
│                                             │
│  4. EXERCISE (5-10 min)                     │
│     You try it. Solution provided.           │
│                                             │
│  5. COMMON MISTAKES (2 min)                 │
│     I show you what breaks, why, and fix.   │
│                                             │
│  6. CHECKPOINT (30 sec)                     │
│     One question. If you can answer, move on.│
│                                             │
└────────────────────────────────────────────┘
```

## The Project: TaskFlow

You will build this feature by feature:

```
Week 1-2:  Single board with hardcoded tasks
Week 3-4:  Add/edit/delete tasks, search, filter
Week 5-6:  Multiple boards, drag-and-drop, data persistence
Week 7-8:  User auth, protected routes, multi-user
Week 9-10: Backend API, real-time sync, teams
Week 11-12: Testing, performance, polish, deploy
```

## Prerequisites

Before starting, make sure you can answer YES to:
- I know HTML (divs, forms, buttons)
- I know basic CSS (flexbox, grid, colors)
- I know basic JavaScript (variables, functions, arrays, objects, if/else, loops)

If "no" to JavaScript, do this first:
→ [JavaScript Crash Course (30 min)](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Guide/Introduction)

## Your Tools

- **Node.js** (v18+) — Download from nodejs.org
- **VS Code** — code editor
- **Terminal** — your command line

## How To Use This Course

1. Read the lesson
2. **Type every line of code yourself** — no copy-paste
3. Run the command, see it work
4. Do the exercise before looking at the solution
5. Intentionally make the mistakes shown (so you recognize them later)
6. Move to next lesson only when you pass the checkpoint
